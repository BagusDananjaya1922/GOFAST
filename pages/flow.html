User Flow Diagram

Dokumentasi logika sistem aplikasi GOFAST yang dipetakan secara menyeluruh dari gerbang masuk akun hingga skenario penyelesaian order di lapangan.

1. Alur Login & Registrasi (Gate)

Pemetaan jalur masuk aplikasi berdasarkan status kepemilikan akun pengguna beserta validasinya.

01
🚀

Start & Onboarding

Penjelasan: Pengguna membuka aplikasi untuk pertama kali dan disambut oleh layar Onboarding untuk pengenalan singkat mengenai utilitas aplikasi GOFAST.

02
⚡

Pendaftaran Akun Baru

Penjelasan: Jika belum punya akun, user diarahkan ke menu Register untuk mengisi formulir pendaftaran, verifikasi email, dan memasukkan kode OTP. Jika gagal, sistem akan mengembalikan ke halaman Register awal.

03
🔑

Autentikasi Login

Penjelasan: Bagi pengguna yang sudah terdaftar, mereka cukup memasukkan kombinasi email dan password. Kesalahan input akan memicu state 'Login Gagal' untuk mencoba kembali.

04
🛠️

Penanganan Lupa Password

Penjelasan: Jika pengguna lupa kata sandi, sistem menyediakan opsi pemulihan mandiri dengan memasukkan konfirmasi email, memverifikasi kode OTP darurat, dan membuat password baru.

2. Alur Proses Pemesanan (Core)

Cabang logika pemesanan armada perjalanan berdasarkan dua mode preferensi yang fleksibel.

01
🎛️

Pemilihan Mode Layanan

Penjelasan: Di halaman Beranda, pengguna dihadapkan pada opsi penentuan jalur efisiensi rute, yaitu memilih antara Mode Cepat atau Mode Normal.

02
🧭

Mode Cepat (Quick Mode)

Penjelasan: Akses instan di mana pengguna langsung melakukan input titik penjemputan sekaligus tujuan, memilih jenis kendaraan, metode bayar, lalu diarahkan ke halaman detail ringkasan pesanan.

03
🗺️

Mode Normal (Standard Mode)

Penjelasan: Jalur komparatif di mana user mengklik pencarian, memilih jenis transportasi, memantau visualisasi informasi kepadatan lalu lintas terlebih dahulu, menentukan titik jemput, baru memproses opsi pembayaran.

04
📡

Pencarian & Alokasi Driver

Penjelasan: Sistem melakukan pinging ke server untuk mencari driver terdekat. Jika driver melakukan pembatalan (cancel), sistem otomatis mengulang proses pencarian (looping) ke driver lainnya.

05
🚘

Dalam Perjalanan

Penjelasan: Status berubah saat driver sukses menjemput pelanggan. Peta interaktif bertransisi ke mode berkendara real-time hingga kendaraan menyentuh koordinat tujuan akhir.

06
🏁

Penyelesaian & Rating

Penjelasan: Perjalanan dinyatakan selesai secara operasional. Pengguna diarahkan ke modul pemberian rating bintang serta ulasan performa sebelum dialihkan kembali ke siklus akhir (End).
